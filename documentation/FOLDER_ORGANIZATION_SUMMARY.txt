✨ RECENT FIXES & IMPROVEMENTS (Session 2026-06-18 - v3 Release):

✅ Audio Features (v3):
   • SPL (Sound Pressure Level) display: ~60-90 dB SPL with room calibration

   • Configurable auto-reduce threshold: Adjustable 0-100% for room-specific calibration
   • Auto-volume reduction: Reduces volume by 5% when sustained above threshold for 2 seconds

✅ Code Architecture:
   • Extracted all UI building into modular components (ui/ folder)
   • Created AudioService for audio analyzer management
   • Refactored main.py from 1100+ lines → cleaner orchestrator
   • Improved separation of concerns (services, workers, UI, widgets, dialogs)

✅ UI/UX Refactoring:
   • Moved playback bar creation to ui/playback_bar.py
   • Moved sidebar creation to ui/sidebar.py
   • Moved page creation (download, pitch, extra) to individual modules
   • Created ui/main_layout.py as orchestrator
   • main.py now only handles event connections and app logic

✅ Settings & Configuration:
   • Added "🔊 Audio Settings" section to settings dialog
   • Settings now include: measurement_mode, auto_reduce_threshold
   • Integrated number-type field support in settings dialog
   • Added QComboBox dropdown selector for measurement modes

✅ File Management:
   • Added config/history.json to .gitignore (local user data only)
   • history.json no longer tracked in version control
   • Ensures clean collaborative development

✅ Stability:
   • Pauses audio analyzer during settings dialog to prevent conflicts
   • Properly resumes analyzer after settings close
   • Improved audio service cleanup on application exit
   • No more UI hangs when switching audio modes

✅ Build System (v3):
   • Version bumped to 3 in build.py and spec file
   • INSTALLATION.txt copied to dist/ root for user visibility
